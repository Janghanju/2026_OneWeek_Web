# 🛠️ 작업 리포트 (Work Report)

## ✅ 완료된 작업

1.  **로컬 개발 환경 복구**
    *   충돌나던 Node.js 프로세스들을 정리했습니다.
    *   **프론트엔드 (Port 3000)**와 **백엔드 (Port 3001)**를 각각 실행하여 정상 작동을 확인했습니다.
    *   이제 로컬에서 `http://localhost:3000`으로 접속하여 개발하실 수 있습니다.

2.  **서버 배포 설정 보안 강화**
    *   `docker-compose.yml`에서 하드코딩된 비밀번호를 제거하고 환경 변수(`${VARIABLE}`)로 대체했습니다.
    *   `.dockerignore` 파일을 추가하여 `.env` 파일 등 민감한 정보가 도커 이미지에 포함되지 않도록 차단했습니다.

3.  **가이드 문서 업데이트**
    *   `DEPLOY_GUIDE.md`: 로컬 개발 방법과 서버 배포 방법을 분리하여 상세히 기술했습니다.
    *   서버 연결 시 필요한 `.env` 설정 방법을 명시했습니다.

## 🚀 사용 방법 요약

### 1. 로컬에서 개발할 때 (지금 상태)
현재 백엔드와 프론트엔드가 실행 중입니다. 브라우저에서 확인해 보세요.
만약 껐다가 다시 켜려면:

*   **터미널 1**: `npm run dev` (프론트엔드)
*   **터미널 2**: `cd backend && npm run start:dev` (백엔드)

### 2. 서버에 배포할 때
서버에 접속해서 다음을 수행하세요. (상세 내용은 `DEPLOY_GUIDE.md` 참고)

1.  `git pull` (최신 코드 받기)
2.  `.env` 파일 생성 및 비밀번호 설정 (한 번만 하면 됨)
3.  `docker-compose up -d --build` (실행)
